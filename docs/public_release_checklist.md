# TRACE Public Release Checklist

Status: draft generated by M1B public prep.

Before publishing:

- Confirm frontend build passes.
- Confirm frontend tests pass.
- Confirm backend pytest passes.
- Confirm real-lab screenshots are sanitized or cropped.
- Do not publish generated report ZIPs.
- Do not publish patch backup folders.
- Do not publish .venv, 
ode_modules, rontend/dist, cache folders, logs, or runtime bundles.
- Do not publish lab passwords or private credentials.
- Do not publish private evidence captures unless sanitized.
- Keep TRACE positioned as read-only IT Ops diagnostics, not a pentest, not a remediation tool.

Recommended GitHub description:

TRACE - local-first, read-only IT Ops diagnostics for Microsoft 365, AD, DNS, SMB, endpoints, and FactoryOps-style support scenarios.
